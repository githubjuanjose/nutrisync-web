NutriSync — Team Testing & Access Guide

For the founding team to try the web apps and install the mobile app on real phones — works across countries (you in Greece, team in Spain), because everything is shared over the internet, not a local network.

Two ways to test the mobile app:

  • A. Expo Go (live dev) — fastest, for day-to-day development. Each person installs one app, scans a QR, and sees changes instantly.
  • B. EAS Build / Update (shareable test builds) — a real installable app (Android .apk / iOS via TestFlight) shared by a link. Best for non-technical testers and stakeholders.

Analogy: Expo Go is like screen-sharing your running app (great while building); EAS Build is like shipping a sealed product sample (great for testers who just want to tap an icon).

0. One-time prerequisites

Everyone (developers):

  1. Install Node.js 18 LTS+ — https://nodejs.org
  2. Install Git — https://git-scm.com
  3. Create a free Expo account — https://expo.dev/signup (one shared org account recommended, e.g. nutrisync)
  4. On their phone, install Expo Go: iOS App Store / Google Play

Testers (non-developers): just need a phone + the link you send them (Section B). Nothing else.

A. Run & share the app live with Expo Go

A1. Run locally (developer)

git clone <your-repo-url> nutrisync
cd nutrisync/apps/mobile
npm install
npx expo start

A QR code appears in the terminal/browser. On the same Wi-Fi, scan it with Expo Go (Android) or the Camera app (iOS) → the app opens.

A2. Share across networks (Greece ↔ Spain) — tunnel mode

Same Wi-Fi isn't possible across countries, so use a tunnel (routes through Expo's servers, no network config):

npx expo start --tunnel

Send the QR or the exp://… link to the team in Spain. They open it in Expo Go and see your running app live, wherever they are. (First run may ask to install @expo/ngrok — accept.)

Note: tunnel needs your laptop running expo start. For always-available testing that doesn't depend on your machine, use B.

B. Shareable test builds with EAS (recommended for testers)

EAS (Expo Application Services) builds the app in the cloud and gives you a link/QR anyone can use — no laptop required, perfect for a distributed team.

B1. One-time setup (developer)

npm install -g eas-cli
eas login                      # use the shared Expo account
cd nutrisync/apps/mobile
eas build:configure            # creates eas.json with build profiles

B2. Android — easiest path (installable APK)

eas build --platform android --profile preview
  • Produces a downloadable .apk + a page with a QR code/link.
  • Send the link to the team. On Android: tap → allow "install from unknown sources" → install. Done.
  • This is the fastest way for Spain testers to get the app on a real phone.

B3. iOS — via TestFlight (Apple requirement)

eas build --platform ios --profile preview
eas submit --platform ios       # uploads to App Store Connect → TestFlight
  • Requires an Apple Developer account ($99/yr) for the org.
  • Add testers' emails as Internal/External testers in TestFlight; they install Apple's TestFlight app and accept the invite.
  • iOS can't sideload like Android, so TestFlight is the standard route.

B4. Instant updates without rebuilding — EAS Update (OTA)

After the first build is installed, push JS/UI changes over-the-air:

eas update --branch preview --message "tweak Home screen"

Testers just reopen the app and get the latest — no reinstall. Use this for rapid iteration with the Spain team.

Suggested eas.json profiles (created by build:configure, tune as needed)

{
  "build": {
    "preview": { "distribution": "internal", "channel": "preview",
      "android": { "buildType": "apk" } },
    "production": { "channel": "production" }
  }
}

C. Access the WEB apps

You have three web surfaces in nutrisync/web/ + nutrisync/prototype/. Two ways to share them:

C1. Open locally (instant, no internet sharing)

Just double-click these files (they need no build step):

  • web/marketing/index.html — marketing site + waitlist
  • web/app/index.html — responsive product web app
  • prototype/index.html — high-fidelity phone prototype
  • web/business-case/index.html — protected business-case site *(placeholder gate)*

C2. Put them online for the team (recommended) — pick ONE

All free and take minutes:

Option 1 — Netlify Drop (no account needed):

  1. Go to https://app.netlify.com/drop
  2. Drag the web/ folder (or each subfolder) onto the page.
  3. You get a public URL like https://nutrisync-app.netlify.app — share it with Spain.

Option 2 — Vercel:

npm i -g vercel
cd nutrisync/web/app
vercel            # follow prompts → returns a shareable URL

Option 3 — GitHub Pages (ties into your repo):

  1. Push the repo to GitHub.
  2. Repo → Settings → Pages → deploy from main /web.
  3. URL: https://<org>.github.io/nutrisync/...

For the business-case site, don't rely on the placeholder password — put real access control in front (Cloudflare Access email allowlist, backlog item H2) before sharing externally.

C3. The web app and mobile app, longer term

At production time the React Native app also runs on web via Expo Web (npx expo start --web), so a single codebase serves iOS, Android, and the responsive web app — this is the "one entry point" in the backlog (Epic F). The standalone HTML web apps here are for immediate review and stakeholder demos.

D. Quick "send to the team in Spain" checklist

  • ☐ Repo pushed to GitHub; invite the team as collaborators.
  • ☐ They install Node, Git, Expo Go, and run npm install in apps/mobile.
  • ☐ For live dev: you run npx expo start --tunnel, share the QR.
  • ☐ For standalone testing: run eas build -p android --profile preview, send the APK link (Android) and/or TestFlight invite (iOS).
  • ☐ Deploy web/ to Netlify/Vercel; share the URLs.
  • ☐ Iterate with eas update (OTA) — no reinstalls.

E. Costs to be aware of

  • Expo account, Expo Go, EAS Update, Netlify/Vercel hobby: free.
  • EAS Build: generous free tier; paid for high build volume.
  • Apple Developer Program: $99/yr (required for any iOS/TestFlight testing).
  • Google Play (only when you publish to the store, not for APK testing): $25 one-time.

F. Troubleshooting

  • QR won't load in Expo Go → use --tunnel; ensure phone has internet.
  • Android "can't install" → enable "install unknown apps" for the browser/Files app.
  • iOS build needs credentials → eas build will prompt to let Expo manage Apple credentials; say yes (needs the Apple Developer account).
  • Different SDK versions → make sure everyone is on the same Expo SDK (this project: SDK 51) to avoid Expo Go mismatches.
